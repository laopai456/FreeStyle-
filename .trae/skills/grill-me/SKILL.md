# Grill Me — 苏格拉底式追问（逆向工程适配版）

## 核心指令

对用户的计划/方案/需求进行**无情的追问**，逐个遍历决策树的每个分支，一次只问一个问题，直到达成共识。

对于每个问题，先给出**我的推荐答案**，然后让用户确认或纠正。

## 何时使用

| 触发条件 | 示例 |
|---------|------|
| 用户说"grill me" | "grill me 关于动态发型方案" |
| 用户要求验证方案 | "帮我确认下这个补丁方案对不对" |
| 用户质疑 AI 理解 | "你确定理解我的需求了吗？" |
| 多方案需要决策 | "Pymem 扫描和 x64dbg 断点哪个先做？" |
| 逆向分析前的需求确认 | "开始逆向之前先对齐一下目标" |

## 规则

### 1. 一次只问一个问题

```
❌ 错误：你的目标是什么？APOLLO 怎么处理？崩溃了吗？
✅ 正确：你认为动态发型的物理效果，最关键的前提条件是哪个？
```

### 2. 每个问题都附上推荐答案

```
问：你认为动态发型的物理效果，最关键的前提条件是哪个？

我的推荐：对象必须是 DDynamicActor 类型（vtable = 0x284AA4C），
因为物理引擎通过虚函数调用 DynamicInit，只有 DDynamicActor 的虚函数表
包含这个入口。SMD 注入骨骼证明了：有骨骼但类型不对 = 无物理效果。

你的看法？
```

### 3. 能从代码库中找到答案的，自己去找

```
❌ 错误：你的游戏基址是多少？
✅ 正确：（自己读 debug_log.md）游戏基址是 0x400000。
```

### 4. 追问直到真正理解

不停在表面回答上，追问背后的原因和约束：

```
用户："我想绕过 APOLLO"
追问："你说的'绕过'具体指什么？"
  A. 完全禁用 APOLLO（风险高，可能封号）
  B. 只绕过 CRC 检测（允许修改代码段）
  C. 在 APOLLO 不检测的范围内操作（只改堆数据）
  D. 其他

我的推荐：C，因为方案 3/4 验证了 WriteProcessMemory 写堆内存不被检测。
```

## 逆向工程专用追问模板

### 需求确认阶段

1. "你的最终目标是什么？（让某个特定发型有物理效果？还是让所有静态发型都能有物理效果？）"
2. "可接受的风险边界在哪？（允许崩溃？允许被检测？允许封号？）"
3. "是否必须在线上环境工作？还是可以离线/私服测试？"

### 技术方案阶段

4. "你倾向于数据驱动（Pymem 扫描定位）还是调试驱动（x64dbg 断点跟踪）？"
5. "对 APOLLO 的容忍度？（完全不碰代码段？还是可以尝试 CRC 绕过？）"
6. "工具选择偏好？（继续用 ctypes 还是迁移到 Pymem？）"

### 执行验证阶段

7. "成功的判断标准是什么？（看到头发摆动？帧率不降？不崩溃运行1小时？）"
8. "失败后的回滚方案？（恢复原始 PAK？重启游戏？）"

## 输出格式

每次追问使用以下格式：

```
### Q{N}: {问题}

**背景**：{为什么问这个}
**推荐答案**：{我的推荐}
**选项**：
- A. {选项A}
- B. {选项B}
- C. {选项C}

请选择或补充。
```
