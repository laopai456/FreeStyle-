# 游戏逆向方法论规则

> 来源: [lbh666/game-re-framework](https://github.com/lbh666/game-re-framework)
> 本地: `FreeStyle/lib/game-re-framework/METHODOLOGY.md`

每条逆向任务开始前，必须先回顾以下原则。

## 核心原则（每次任务前 review）

1. **横向优先于纵向** — 排除一个函数后先看同层兄弟函数，不深入也不往上游追
2. **先枚举再深入** — 进入函数分析前，列出同层所有调用并标注角色
3. **分析一步验证一步** — 推断→调试→推断→调试交替，不连续猜测
4. **双路并行** — 自顶向下（调用链）和自底向上（特征搜索）同时进行
5. **两次排除即跳出** — 同一函数排除2个候选后，必须横向检查同层
6. **"不触发"是最强线索** — 断点不触发→追问走了哪条路径
7. **写补丁前纸面验证完整性** — 不要边写边修

## 特征指令预扫描（替换字节扫描）

**原则：搜索 CODE 中引用已知 DATA 的指令，比搜索 DATA 字节本身可靠。**

| 目标 | 搜索什么 |
|------|---------|
| 魔数比较（如 SSKF） | 引用数据地址的指令（push offset / cmp [mem], addr），不扫数据字节 |
| 时机判定 | COMISS / UCOMISS |
| 随机判定 | IMUL + 常数 → SHR |
| 距离判定 | sqrt + COMISS |
| vtable 调用 | CALL [RAX+offset] |

## 策略切换触发器

| 触发条件 | 动作 |
|---------|------|
| 同一函数排除2个候选 | 跳出，横向检查 |
| 自顶向下连续3轮无进展 | 切到特征搜索 |
| 断点目标场景不触发 | 查另一侧分支路径 |
| 继续两次补丁无效 | 重新审视方向 |

## 反模式

- 一条路卡住后不切换（如扫 SSKF 字节 16KB 找不到 → 应切 ReadFile hook 或引用扫描）
- 强制跳过中间代码
- 在 call 指令前后 hook
- 连续猜测多步不验证
