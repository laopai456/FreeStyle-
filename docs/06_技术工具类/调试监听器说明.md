# 调试监听器说明

## 一、功能概述

调试监听器用于记录用户的完整操作流程，方便分析程序逻辑和调试问题。

## 二、日志文件位置

```
[程序运行目录]/logs/operation_debug.log
```

例如：`D:\py\反编译\FS服装搭配专家v1.0\bin\Debug\logs\operation_debug.log`

## 三、已插入监听的位置

### 1. 主窗体初始化
- **位置**: `MainWindow()` 构造函数
- **记录内容**: 初始化调试器，创建日志会话

### 2. 查看道具
- **位置**: `MainWindow.dgvItemShop_CellContentClick`
- **记录内容**:
  - UI事件点击
  - 道具信息（pakIconNum, pngItemCode, pakItemNum, strItemName）
  - 复制文件操作

### 3. 设为变更前
- **位置**: `ViewOneIconFrm2.btnSetBefore_Click`
- **记录内容**: UI事件点击

### 4. 设为变更后
- **位置**: `ViewOneIconFrm2.btnSetAfter_Click`
- **记录内容**: UI事件点击

### 5. 复制和解包item pak
- **位置**: `MainWindow.bw_DoWorkOtherItem`
- **记录内容**:
  - 操作类型（设置变更前/后）
  - pakItemNum
  - 是否它服
  - 文件复制操作

### 6. 设置变更前后信息
- **位置**: `MainWindow.bw_CompletedWorkOtherItem`
- **记录内容**:
  - tsbBeforeInfo 或 tsbAfterInfo 的完整内容
  - 操作完成状态

### 7. 执行变更
- **位置**: `MainWindow.btnUpdate_Click`
- **记录内容**:
  - UI事件点击
  - tsbBeforeInfo 和 tsbAfterInfo
  - 对话框标题、内容和结果

### 8. 变更核心逻辑
- **位置**: `MainWindow.bw_DoWorkLast`
- **记录内容**:
  - 解析出的所有变量（strAfterPak, strAfterMod, strBeforePak, strBeforeMod等）
  - 文件复制操作
  - 命令执行

## 四、日志格式说明

```
[时间] [类型] 内容 (耗时)
```

### 日志类型

| 类型 | 说明 |
|------|------|
| START | 操作开始 |
| STEP | 操作步骤 |
| END | 操作结束 |
| VAR | 变量值 |
| FILE | 文件操作 |
| CMD | 命令执行 |
| UI | 用户界面事件 |
| DIALOG | 对话框 |
| ERROR | 错误信息 |

## 五、使用方法

### 查看日志
直接打开 `logs/operation_debug.log` 文件即可查看完整操作记录。

### 清空日志
删除 `logs/operation_debug.log` 文件即可清空日志。

### 禁用监听
如需禁用调试监听，注释掉 `MainWindow` 构造函数中的：
```csharp
Elena.Debugger.OperationDebugger.Initialize();
```
