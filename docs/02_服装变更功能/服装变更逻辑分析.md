# FS服装搭配专家 - 服装变更逻辑完整分析

## 一、核心流程概览

```
用户操作流程:
1. 点击表格"查看"按钮 → 打开图标预览窗体
2. 在预览窗体点击"设为变更前"或"设为变更后" → 设置变更前后道具
3. 点击"变更"按钮 → 执行文件替换和重新打包
4. 完成(或点击"还原全部"还原)
```

---

## 二、详细步骤分析

### 步骤1: 查看道具图标

**触发位置**: 表格列 ColumnIndex 1(本服) 或 ColumnIndex 3(它服)

**代码位置**: `UI/Windows/MainWindow.xaml.cs` - `dgvItemShop_CellContentClick`

**操作流程**:
```
1. 点击表格"查看"按钮
   ↓
2. 复制pak文件到cookies目录
   File.Copy([游戏目录]/[pak文件], cookies/[pak文件])
   ↓
3. 执行解包(如果目录不存在)
   bw_DoWorkIcon:
     "pack\resources cookies\[pak文件] -all"
   ↓
4. 显示ViewOneIconFrm2预览窗体
```

**关键代码**:
```csharp
// 获取道具信息
pakIconNum = PakNum.Replace("*", "icon")  // 例如: icon*.pak
pakItemNum = PakNum.Replace("*", "item")  // 例如: item*.pak

File.Copy([游戏目录]/icon*.pak, cookies/icon*.pak, true)

// 解包pak文件
"pack\resources cookies\icon*.pak -all"
```

### 步骤2: 选择变更前道具

**触发位置**: ViewOneIconFrm2窗体 - "设为变更前"按钮 (btnSetBefore)

**代码位置**: `ViewOneIconFrm2.cs` - `btnSetBefore_Click`

**操作流程**:
```
1. 点击"设为变更前"按钮
   ↓
2. 关闭预览窗体
   ↓
3. 调用MainWindow的RefreshControl方法
   myRefresh(道具名, 图标文件, isbefore=true)
   ↓
4. RefreshControl执行:
   ├─ 设置 thisIsbefore = true
   ├─ 如果图标文件存在,显示在picBeforeImg
   └─ 复制item pak并解包(BackgroundWorker异步)
      File.Copy([游戏目录]/item*.pak, cookies/item*.pak)
      "pack\resources cookies\item*.pak -all"
   ↓
5. 解包完成后设置tsbBeforeInfo
   tsbBeforeInfo.Text = "道具名-pak:item*.pak,mod:i12345.bml,eff:特效编号"
```

**关键代码**:
```csharp
// ViewOneIconFrm2中
private void btnSetBefore_Click(object sender, EventArgs e)
{
    base.Close();
    myRefresh(this.thispropname, "u" + this.labCoe.Text + ".png", true);
}

// MainWindow中
private void RefreshControl(string otherPropName, string otherIconPng, bool isbefore)
{
    thisIsbefore = isbefore;
    thisOtherPropName = otherPropName;

    // 如果图标文件已存在,直接显示
    if (File.Exists(cookies/.../u12345.png))
    {
        picBeforeImg.Image = Image.FromFile(...);
    }

    // 解包item pak
    File.Copy([游戏目录]/item*.pak, cookies/item*.pak);

}

// bw_CompletedWorkOtherItem (解包完成后)
if (thisIsbefore)
{
    tsbBeforeInfo.Text = string.Format(
        "{0}-pak:{1},mod:{2},eff:{3}",
        thisOtherPropName,      // 道具名
        pakItemNum,           // item*.pak
        pngItemCode.Replace("u", "i").Replace("png", "bml"),
        effectCode            // 特效编号或"无"
    );
}

**触发位置**: ViewOneIconFrm2窗体 - "设为变更后"按钮 (btnSetAfter)

**代码位置**: `ViewOneIconFrm2.cs` - `btnSetAfter_Click`

**操作流程**:
```
1. 点击"设为变更后"按钮
   ↓

### 步骤4: 执行变更操作

**触发位置**: MainWindow - "变更"按钮 (btnUpdate)

**代码位置**: `UI/Windows/MainWindow.xaml.cs` - `btnUpdate_Click` 和 `bw_DoWorkLast`

**操作流程**:
```
1. 点击"变更"按钮
   ↓
2. 验证条件:
   ├─ 变更前已选择 (tsbBeforeInfo.Text != "beforeInfo")
   ├─ 变更后已选择 (tsbAfterInfo.Text != "afterinfo")
   └─ 变更前后道具不同
   ↓
3. 用户确认对话框
   MessageBox.Show("确定将[变更前]变更为[变更后]吗?")
   ↓
4. 执行变更 (bw_DoWorkLast异步)
```

**变更详细步骤** (bw_DoWorkLast):
```
4.1 解析变更前后信息
    strAfterPak = item*.pak  (从tsbAfterInfo解析)
    strAfterMod = i12345.bml (从tsbAfterInfo解析)
    strAfterEff = 特效编号    (从tsbAfterInfo解析)
    strBeforePak = item*.pak (从tsbBeforeInfo解析)
    strBeforeMod = i12345.bml (从tsbBeforeInfo解析)
    strBeforeEff = 特效编号   (从tsbBeforeInfo解析)

4.2 复制变更后的文件到变更前位置
    source = cookies/i12345.bml (变更后的模块文件)
    target = cookies/i12345.bml (变更前的模块文件)
    File.Copy(source, target, true)

4.3 重新打包item pak
    "pack\resources -file2pak cookies\item* [游戏目录]\item*.pak"

4.4 处理res.pak (如果变更后来自它服且有res.pak)
    4.4.1 比较res.pak大小
         MyRespakSize vs OtherRespakSize

    4.4.2 如果大小不同且<200KB
         ├─ 复制res.pak到cookies
            File.Copy([它服]/res*.pak, cookies/res*.pak)

         ├─ 解包res.pak
            "pack\resources cookies\res*.pak -all"

         ├─ 复制目录
            CopyDirectory(cookies\item*, othercookies\)

         ├─ 打包到游戏目录
            "pack\resources -file2pak othercookies\item* [它服]/res*.pak"

    4.4.3 如果已经解包过
         File.Copy(othercookies/res*.pak, [它服]/res*.pak)

4.5 处理特效 (如果前后都有特效且不同)
    4.5.1 显示确认对话框
         "是否将特效:[旧]变更为[新]?"

    4.5.2 如果确认:
         ├─ 修改itemshop.txt
            替换: ItemCode\tPakNum\t旧特效\t
            为:    ItemCode\tPakNum\t新特效\t
            文件: cookies\item_text_pak\itemshop.txt

         └─ 重新打包item_text_pak
            "pack\resources -file2pak cookies\item_text_pak [游戏目录]\item_text.pak"
```

**关键代码**:
```csharp
private void bw_DoWorkLast(object sender, DoWorkEventArgs e)
{
    // 1. 解析信息
    strAfterPak = tsbAfterInfo.Text.Split(',')[0].Split(':')[1];
    strAfterMod = tsbAfterInfo.Text.Split(',')[1].Split(':')[1];
    strBeforePak = tsbBeforeInfo.Text.Split(',')[0].Split(':')[1];
    strBeforeMod = tsbBeforeInfo.Text.Split(',')[1].Split(':')[1];

    // 2. 复制文件
    source = Environment.CurrentDirectory + "\\" +
             (tsbAfterInfo.Contains("server:me") ? "cookies" : "othercookies") +
             "\\" + strAfterPak + "\\" + strAfterMod;
    target = Environment.CurrentDirectory + "\\cookies\\" + strBeforePak + "\\" + strBeforeMod;
    File.Copy(source, target, true);

    // 3. 重新打包
    "pack\\resources -file2pak cookies\\[strBeforePak] [游戏目录]\\[strBeforePak]";

    // 4. 处理res.pak (如果是它服)
    if (tsbAfterInfo.IndexOf("server:me") == -1)
    {
        // ... res.pak处理逻辑
    }

    // 5. 处理特效
    if (strBeforeEff != "" || strAfterEff != "")
    {
        // ... 特效处理逻辑
    }
}
```

### 步骤5: 全部还原

**触发位置**: MainWindow - "还原全部"按钮 (btnReSetAll)

**代码位置**: `UI/Windows/MainWindow.xaml.cs` - `btnReSetAll_Click`

**操作流程**:
```
1. 点击"还原全部"按钮
   ↓
2. 确认对话框
   MessageBox.Show("您确定要还原所有道具吗?")
   ↓
3. 执行T2REPAIR.exe
   Process.Start("T2REPAIR.exe", 工作目录=游戏目录)
   ↓
4. 完成
```

**关键代码**:
```csharp
private void btnReSetAll_Click(object sender, EventArgs e)
{
    if (MessageBox.Show("您确定要还原所有道具吗？", "确定",
        MessageBoxButtons.OKCancel) != DialogResult.Cancel)
    {
        Process process = new Process();
        process.StartInfo.FileName = "cmd.exe";
        process.StartInfo.WorkingDirectory = this.strInstallDirectory;
        process.Start();
        process.StandardInput.WriteLine("T2REPAIR.exe");
        process.StandardInput.WriteLine("exit");
    }
}
```

## 四、核心工具命令

### 解包命令
```bash
# 解包整个pak
pack\resources "[pak文件路径]" -all

# 只解包png图片
pack\resources "[pak文件路径]" -byname .+\.png
```

### 打包命令
```bash
# 打包目录到pak
pack\resources -file2pak "[目录路径]" "[pak文件路径]"
```

## 六、批量变更需求分析

### 当前限制
- 每次只能单对单变更(一个变更前 → 一个变更后)
- 需要手动选择每一个道具
- 变更前后都要手动查看和设置

### 批量变更可能的需求方向
1. **按列表批量变更**: 读取配置文件,自动执行多组变更
2. **批量复制特效**: 将一个道具的特效批量应用到多个道具
3. **批量替换模型**: 将一个模型批量替换到多个道具
4. **导出/导入配置**: 保存和加载变更方案

### 批量变更的关键点
- 需要批量读取变更前后的配置
- 需要循环执行"复制文件 → 打包"流程
- 需要处理res.pak和特效的批量逻辑
- 需要进度显示和错误处理

## 八、注意事项

1. **必须先关闭游戏**: 变更前需要先关闭游戏
2. **文件锁定**: pak文件被游戏占用时无法修改
3. **异步操作**: 解包和打包使用BackgroundWorker异步执行
4. **错误处理**: 大部分操作都有try-catch错误处理
5. **进度显示**: 解包大文件时显示进度提示
6. **res.pak特殊处理**: 只在它服变更时需要处理res.pak
7. **特效确认**: 有特效变更时需要用户二次确认
