# 动态主题背景上传功能设计文档

## 功能目标

在皮肤设置窗口中增加“导入动态背景”功能，让用户选择本地视频文件后，工具自动：

1. 复制视频到程序 `videos/` 目录
2. 创建一个新的动态视频主题
3. 写入 `skins/{主题ID}/theme.json`
4. 刷新主题列表
5. 让用户可以直接应用并欣赏动态背景

当前底层视频播放已经存在，缺的是上传/导入 UI 和自动生成主题逻辑。

---

## 当前已有能力

### 主题模型

文件：`Core/Models/SkinTheme.cs`

`BackgroundStyle` 已有字段：

```csharp
public string Type { get; set; } = "gradient";
public string? VideoPath { get; set; }
public double Volume { get; set; } = 0;
```

### 主题加载与保存

文件：`Core/Services/ThemeLoader.cs`

已有能力：

- `LoadAllThemes()`：扫描 `skins/*/theme.json`
- `SaveTheme(SkinTheme theme)`：保存主题 JSON
- `EnsureDefaultThemesExist()`：生成暗黑主题

### 视频播放

文件：`UI/Windows/MainWindow.xaml.cs`

已有能力：

- `ApplyVideoBackground(SkinTheme theme)`
- `EnsureVLCInitialized()`
- `OnVideoFormat()` / `OnLockVideo()` / `OnUnlockVideo()`
- `VideoBackgroundImage` 底层图片控件渲染视频帧

### 主题选择窗口

文件：`UI/Windows/SkinWindow.xaml(.cs)`

已有能力：

- 显示主题列表
- 预览主题
- 应用主题

缺少：

- 导入视频按钮
- 复制视频到 `videos/`
- 自动创建视频主题
- 刷新主题列表

## 推荐用户流程

```text
打开主窗口
→ 点击右上角 🎨 换肤
→ 点击“导入动态背景”
→ 选择本地 .mp4 视频
→ 工具复制视频到 videos/custom_时间戳.mp4
→ 工具生成 skins/custom_video_时间戳/theme.json
→ 主题列表自动刷新并选中新主题
→ 点击“应用”
→ 主窗口立即播放该视频背景
```

## 文件命名规则

为避免覆盖用户文件，使用时间戳：

```text
videos/custom_20260605_153012.mp4
skins/custom_video_20260605_153012/theme.json
```

主题字段：

```json
{
  "id": "custom_video_20260605_153012",
  "name": "动态背景 20260605 153012",
  "author": "User",
  "version": "1.0",
  "description": "用户导入的视频动态背景",
  "styles": {
    "window": {
      "background": {
        "type": "video",
        "videoPath": "custom_20260605_153012.mp4",
        "volume": 0
      }
    }
  }
}
```

## 实现步骤

### 1. 修改 `SkinWindow.xaml`

增加按钮：

```xml
<Button Content="导入动态背景"
        Width="130"
        Height="40"
        Style="{StaticResource GlassButton}"
        Click="ImportVideoTheme_Click" />
```

### 2. 修改 `SkinWindow.xaml.cs`

新增事件：

```csharp
private async void ImportVideoTheme_Click(object sender, RoutedEventArgs e)
```

逻辑：

1. `OpenFileDialog` 选择视频
2. 校验扩展名和文件存在
3. 创建 `videos/` 目录
4. 异步复制视频
5. 创建 `SkinTheme`
6. 调用 `skinManager.AddCustomTheme(theme)` 或 `ThemeLoader.SaveTheme(theme)`
7. 调用 `skinManager.LoadThemes()`
8. 刷新 `lstSkins.ItemsSource`
9. 自动选中新主题

### 3. 修改 `SkinManager.cs`（可选）

增加包装方法：

```csharp
public void ReloadThemes()
{
    LoadThemes();
}
```

这样 UI 层不用直接关心加载细节。

### 4. 日志与提示

建议每一步输出：

```text
[SkinWindow] 开始导入动态背景: sourcePath
[SkinWindow] 视频复制完成: destPath
[SkinWindow] 主题创建完成: theme.Id
[SkinWindow] 刷新主题列表完成
```

失败时提示用户，不要崩溃。

## 格式支持建议

第一版只开放：

```text
*.mp4
```

原因：

- README 当前也写 mp4
- 兼容性最好
- 降低用户导入失败概率

后续可以开放：

```text
*.mp4;*.mkv;*.avi;*.webm;*.mov
```

但要在提示里注明：虽然 LibVLC 能解码很多格式，但并不保证每个视频都能稳定作为背景。

## 风险与处理

| 风险 | 处理 |
|------|------|
| 视频文件太大导致 UI 卡死 | 使用 `Task.Run` 异步复制 |
| 文件名重复 | 时间戳命名 |
| 视频不存在或复制失败 | try/catch 提示错误 |
| 主题 JSON 创建失败 | 不刷新列表，显示失败原因 |
| 用户导入无效格式 | OpenFileDialog 过滤 + 扩展名校验 |
| 切换视频主题内存泄漏 | 复用现有 `ApplyVideoBackground()` 停止旧播放器逻辑，后续重点测试 |

## 测试用例

1. 导入一个小 MP4，确认主题出现并能应用
2. 导入同名 MP4 两次，确认不会覆盖旧主题
3. 导入大视频，确认 UI 不长时间假死
4. 选择后取消，确认没有创建空主题
5. 删除/移动视频后启动，确认程序不崩溃，只提示视频不存在
6. 从视频主题切回渐变主题，确认视频停止并隐藏
7. 多次切换视频主题，观察内存和 CPU 是否异常上涨

---

## 当前结论

底层视频播放能力已经完成，动态主题背景上传属于低风险增量功能。建议优先实现最小闭环：

```text
导入 mp4 → 复制 videos → 生成 theme.json → 刷新列表 → 应用播放
```

后续再加删除主题、视频预览、格式扩展等增强功能。
