# 动态发型功能需求文档

## 一、项目背景

### 1.1 用户需求
用户希望将游戏中的静态发型（无物理摆动效果）替换为动态发型（具有物理摆动效果），以提升游戏视觉体验。

### 1.2 历史研究回顾

#### 初期错误假设（已否定）
假设：EffectCode 决定动态效果

分析结果：
- EffectCode 存储在 itemshop.txt 中
- 特效是独立的后期添加效果
- 服装变更逻辑显示特效可独立修改，不直接影响模型物理特性

**结论**：EffectCode 与发型动态效果无关

#### 正确研究方向
- 动态发型的物理模拟数据编码在 bml 模型文件内部
- 需要分析 bml 文件结构，找出物理模拟参数

## 二、技术分析

### 2.1 已知资源包结构

| Pak 文件 | 内容 | 说明 |
|---------|------|------|
| item*.pak | BML 模型文件 | 3D 模型数据，包含骨骼和可能的物理参数 |
| res*.pak | PNG 贴图资源 | 纹理贴图 |
| icon*.pak | PNG 图标资源 | UI 图标 |

### 2.2 BML 文件初步分析

已知 BML 文件示例：
- i50119961.bml（动态发型，最强超赛发型）：1255 字节
- i50120891.bml（静态发型，玲珑水晶发型）：1591 字节

**观察**：静态发型文件反而更大，说明文件大小与动态效果无关。

### 2.3 需要深入研究的内容

#### 2.3.1 BML 文件格式分析
- 文件头结构（魔数、版本信息）
- 数据块类型和数量
- 骨骼数据结构
- 物理模拟参数（弹簧、重力、阻尼等）

#### 2.3.2 动态发型特征识别
- 动态发型 BML 的独特数据块
- 物理模拟参数的存储位置
- 与静态发型 BML 的关键差异

#### 2.3.3 替换可行性分析
- 是否可以直接复制动态发型 BML 替换静态发型
- 需要调整哪些参数以适配不同发型模型
- 替换后是否会导致游戏崩溃或显示异常

## 三、功能需求

### 3.1 核心功能
将用户指定的静态发型道具替换为动态发型道具。

### 3.2 用户操作流程（期望）
```
1. 选择目标静态发型道具
   ↓
2. 选择源动态发型道具
   ↓
3. 点击"替换"按钮
   ↓
4. 系统自动：
   - 复制动态发型的 BML 文件
   - 调整物理模拟参数（如需要）
   - 重新打包 item*.pak
   ↓
5. 完成提示
```

### 3.3 功能范围
- **包含**：静态发型 → 动态发型替换
- **暂不包含**：动态发型参数调整、发型组合编辑

## 四、技术方案

### 4.1 研究阶段

#### 阶段 1：样本收集
- [ ] 收集至少 5 个动态发型样本
- [ ] 收集至少 5 个静态发型样本
- [ ] 记录每个样本的 ItemCode、PakNum、BML 文件路径

#### 阶段 2：格式分析
- [ ] 解析 BML 文件头结构
- [ ] 识别所有数据块类型
- [ ] 分析骨骼数据结构
- [ ] 定位物理模拟参数位置

#### 阶段 3：模式识别
- [ ] 对比动态发型和静态发型的 BML 差异
- [ ] 找出动态发型独有的数据块或参数
- [ ] 建立动态发型特征识别规则

#### 阶段 4：替换测试
- [ ] 小规模测试：复制动态发型 BML 替换静态发型
- [ ] 测试结果记录：是否成功、是否有异常
- [ ] 如失败，分析失败原因并调整方案

#### 阶段 5：工具开发（如替换成功）
- [ ] 开发 BML 文件分析工具
- [ ] 开发自动替换工具
- [ ] 集成到 FS 服装搭配专家主程序

### 4.2 预期技术难点

#### 难点 1：BML 文件格式未知
- **风险**：没有公开的 BML 格式文档
- **应对**：通过逆向工程分析文件结构
- **工具**：十六进制编辑器（HxD）、二进制分析工具

#### 难点 2：物理模拟参数复杂性
- **风险**：物理参数可能与骨骼绑定，调整困难
- **应对**：尝试直接复制整个物理参数块
- **备选**：如果调整失败，只做简单替换，接受物理效果可能不完美

#### 难点 3：游戏兼容性
- **风险**：替换后游戏可能崩溃
- **应对**：
  - 备份原文件
  - 先在小范围测试
  - 提供"还原全部"功能
