#### 3. BML文件格式重大突破（关键进展！）

**源代码分析**：
- 分析了`FileSys.cpp`中的`LoadBinaryXML`函数
- 发现BML文件是XML格式，有两种存储方式：
  1. XOR加密格式：每个字节与0xFF进行XOR运算
  2. 纯文本XML格式：直接存储的XML

**解码测试**：
- 开发了Python解码工具`bml_decoder.py`
- 成功解码`i50125001.bml`（静态发型）：1605 bytes
- 发现`i50125031.bml`（动态发型）：1582 bytes，已经是纯XML格式

**XML结构解析**：
```xml
<?xml version="1.0"?>
<root>
   <channel>1</channel>
   <character type="1">
      <object>
         <type>normal</type>
         <mesh>res764\i50125031_MT.smd</mesh>
         <texture>res764\i50125031.png</texture>
      </object>
      <!-- 每个道具最多5个object -->
   </character>
   <!-- type=1,2,3,4,5,6 对应不同角色类型 -->
</root>
```

#### 4. BML文件分析结果

**BML文件包含的信息**：
- ✅ channel（装备位掩码）
- ✅ character type（角色类型）
- ✅ object type（normal/fskin/uskin/lskin）
- ✅ mesh文件（SMD模型路径）
- ✅ texture文件（纹理路径）

**BML文件不包含的信息**：
- ❌ 物理模拟参数（重力、弹性、阻尼等）
- ❌ 骨骼绑定信息
- ❌ 动态效果标志位
- ❌ 物理模拟开关

**核心结论**：
- **BML文件只存储道具的基本信息和资源路径**
- **物理模拟参数不存储在BML文件中**
- **物理参数最可能存储在SMD模型文件中**

### 使用工具分析SMD文件

#### 使用HxD分析SMD
```
1. 启动 HxD
2. File → Open
3. 选择 SMD 文件（如：i50125031_MT.smd）
4. 查看：
   - 文件头（前几行）
   - 数据块结构
   - 文本数据（可能的XML或ASCII格式）
5. 使用Ctrl+F搜索关键词：
   - "bone"
   - "physics"
   - "gravity"
   - "mass"
```

#### 使用Blender导入SMD
```
1. 启动 Blender
2. File → Import → Valve Source SMD (.smd)
3. 选择 SMD 文件（如：i50125031_MT.smd）
4. 查看模型：
   - 按 Z 键切换线框/实体模式
   - 按 1/2/3 切换顶点/边/面编辑
   - 按 Ctrl+Tab 切换物体/编辑/纹理绘制模式
5. 查看骨骼：
   - 在右侧 Outliner 面板查看 Armature
   - 展开查看骨骼列表
   - 选择骨骼查看属性
```
