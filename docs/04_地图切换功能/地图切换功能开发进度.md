## 核心逻辑

### 地图加载流程
```
LoadMapsFromLibrary(libraryPath)
1. 扫描指定目录下的所有子文件夹
2. 检查每个文件夹是否包含 stage02.pak
3. 查找文件夹内的第一个 PNG/JPG 文件作为预览图
4. 创建 MapM 对象并添加到列表
5. 绑定到 DataGridView
```

### 交互流程
```
用户操作: 选择地图库目录 → 自动扫描 → 显示地图列表
用户操作: 点击地图行 → 右侧显示预览图和详细信息
用户操作: 输入搜索关键词 → 过滤显示匹配的地图
```

## 已修复问题

### BatchSelectFrm chkSelectAll NullReferenceException
- **文件**: `BatchSelectFrm.Designer.cs`
- **问题**: chkSelectAll 复选框声明但未实例化,运行时报 NullReferenceException
- **错误信息**: 对象引用未设置为对象的实例
- **解决**: 在 InitializeComponent() 中添加 `this.chkSelectAll = new System.Windows.Forms.CheckBox();` 并配置属性
- **注意**: 问题曾反复出现,需确保修改已保存

### BatchSelectFrm 界面布局重叠问题 (第1次修复)
- **文件**: `BatchSelectFrm.Designer.cs`
- **问题**: chkSelectAll 位置与 lblTitle 过近
- **修复**: 调整 chkSelectAll 位置从 (250, 485) 到 (150, 485), lblTitle 文本改为 "已选择 0 项"

### BatchSelectFrm 窗口标题与表头重合问题 (第2次修复)
- **文件**: `BatchSelectFrm.Designer.cs`
- **问题**: DataGridView表头（图标、序号列）与窗口标题重合
- **原因**: DataGridView的Location(12, 12)太高,与窗口标题栏冲突
- **修复**: 调整DataGridView位置和大小
  - Location: (12, 12) → (12, 40), 向下移动28像素
  - Size: (960, 450) → (960, 420), 高度减少30像素

### BatchSelectFrm 控件布局优化 (第3次修复)

### DataGridView 类型转换错误
- **问题**: DataGridView 的 Image 列 (colPreview) 绑定字符串路径导致 System.FormatException
- **错误信息**: 从 'System.String' 到 'System.Drawing.Image' 的强制转换无效
- **解决**: 移除表格中的预览列 (colPreview), 只保留右侧独立预览图

### colPreview 列未找到错误
- **问题**: dgvMap_CellClick 事件中通过 row.Cells["colPreview"] 访问已删除的列
- **解决**: 直接从数据源 (MapM 对象) 获取数据,使用 row.DataBoundItem

### MapControl ContextMenuStrip 类型转换错误
- **文件**: `controls/MapControl.Designer.cs`
- **问题**: 尝试对ContextMenuStrip调用BeginInit/EndInit,但ContextMenuStrip不支持ISupportInitialize接口
- **错误信息**: 无法将类型为 "System.Windows.Forms.ContextMenuStrip" 的对象强制转换为类型 "System.ComponentModel.ISupportInitialize"
- **解决**: 移除 `((System.ComponentModel.ISupportInitialize)(this.cmsPreview)).BeginInit()` 和对应的EndInit调用
- **技术说明**: 只有DataGridView、PictureBox等少数控件支持ISupportInitialize,ContextMenuStrip不支持
