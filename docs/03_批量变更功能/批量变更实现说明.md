## 六、道具类型匹配规则

根据pak编号范围判断道具类型（可根据实际情况调整）：

| pak编号范围 | 道具类型 |
|------------|----------|
| 1-500 | 上衣 |
| 501-1000 | 裤子 |
| 1001-1500 | 鞋子 |
| 1501-2000 | 发型 |
| 2001-2500 | 面部 |
| 其他 | 其他 |

**匹配逻辑**：
- 变更前道具类型 == 变更后道具类型 → 匹配成功
- 每个变更后道具只能被匹配一次
- 如果类型不匹配，不会进行变更

### 8.2 批量变更的目录依赖问题

**问题根源**：
- 批量选择功能直接读取表格数据，跳过了"选择"按钮的解包步骤
- 如果用户没有先点击"选择"按钮预览道具，对应的pak目录就不会解包
- 批量变更时复制bml文件会失败：找不到路径

**解决方案**：
在`ExecuteSingleUpdate`方法中，添加`EnsurePakUnpacked`辅助方法，在复制bml文件前自动检查并解包所需的pak目录。

**实现代码**：
```csharp
// 辅助方法：确保pak目录已解包
private void EnsurePakUnpacked(string pakName, bool isOtherServer)
{
    string cookiesDir = isOtherServer ? "othercookies" : "cookies";
    string pakPath = Environment.CurrentDirectory + "\\" + cookiesDir + "\\" + pakName;
    string unpackedDir = Environment.CurrentDirectory + "\\" + cookiesDir + "\\" + pakName.Replace(".", "_");

    // 检查解包目录是否存在
    if (!Directory.Exists(unpackedDir))
    {
        // 从游戏目录复制pak文件到cookies目录
        string gameDir = isOtherServer ? this.txtOtherServerPath.SkinTxt.Text : this.strInstallDirectory;
        string sourcePak = gameDir + "\\" + pakName;
        File.Copy(sourcePak, pakPath, true);

        // 解包pak文件
        string unpackCmd = $"pack\\resources \"{pakPath}\" -all";
        conmon.RunCmd(unpackCmd);
    }
}

// 在ExecuteSingleUpdate中调用
EnsurePakUnpacked(beforePakName, false);  // 变更前pak
EnsurePakUnpacked(afterPakName, isAfterOtherServer);  // 变更后pak
```

### 8.3 使用conmon.RunCmd避免打包卡死

**问题背景**：
- 最初使用`Process.Start("cmd.exe", $"/c \"{packCmd}\"")`执行打包命令
- `pack\resources`命令执行后会显示"按任意键关闭"
- 程序会一直等待用户按键，导致界面loading卡死

**修复方案**：
改用原代码的`conmon.RunCmd`方法，该方法会自动添加`&exit`命令避免等待。

**实现代码**：
```csharp
// 之前的方式（会导致卡死）
Process.Start("cmd.exe", $"/c \"{packCmd}\"").WaitForExit();

// 修复后的方式（使用conmon.RunCmd）
conmon.RunCmd(packCmd);
```

**conmon.RunCmd内部实现**（conmon.cs 第264-284行）：
```csharp
public static void RunCmd(string str)
{
    Process process = new Process();
    process.StartInfo.FileName = "cmd.exe";
    process.StartInfo.UseShellExecute = false;
    process.StartInfo.RedirectStandardInput = true;
    process.StartInfo.RedirectStandardOutput = true;
    process.StartInfo.RedirectStandardError = true;
    process.StartInfo.CreateNoWindow = true;
    process.StartInfo.Verb = "RunAs";
    process.Start();

    // 关键：自动添加&exit避免等待按任意键
    process.StandardInput.WriteLine(str + "&exit");
    process.StandardInput.AutoFlush = true;
    process.StandardInput.WriteLine("exit");

    string text = process.StandardOutput.ReadToEnd();
    process.WaitForExit();
    process.Kill();
    process.Dispose();
}
```

**关键点**：
- `RedirectStandardInput = true`：重定向输入流
- `CreateNoWindow = true`：不创建窗口
- `str + "&exit"`：在命令后自动添加退出指令
- `process.StandardOutput.ReadToEnd()`：读取输出确保命令完成

### 8.4 打包命令路径引用问题

**问题背景**：
- 游戏安装路径包含空格：`C:\Program Files (x86)\T2CN\街头篮球\`
- 打包命令未对路径加引号，导致命令行参数解析错误
- pak文件无法正确写入游戏目录，变更无效

**错误示例**：
```csharp
// 错误：路径中的空格会导致参数解析错误
string packCmd = $"pack\\resources -file2pak cookies\\{strBeforePak} {this.strInstallDirectory}\\{strBeforePak.Replace("_", ".")}";
```
执行时会被解析为：
```
pack\resources -file2pak cookies\item747_pak C:\Program Files (x86)\T2CN\街头篮球\item747.pak
                                                    ↑ 这里被切分成了多个参数
```

**修复方案**：
给所有路径参数添加引号：

```csharp
// 正确：使用引号包裹路径
string packCmd = $"pack\\resources -file2pak \"cookies\\{strBeforePak}\" \"{this.strInstallDirectory}\\{strBeforePak.Replace("_", ".")}\"";
```

**修复位置**：
1. **第2126行**：重新打包item pak文件
   ```csharp
   string packCmd = $"pack\\resources -file2pak \"cookies\\{strBeforePak}\" \"{this.strInstallDirectory}\\{strBeforePak.Replace("_", ".")}\"";
   ```

2. **第2157行**：重新打包item_text.pak文件
   ```csharp
   string packTextCmd = $"pack\\resources -file2pak \"cookies\\item_text_pak\" \"{this.strInstallDirectory}\\item_text.pak\"";
   ```

**关键点**：
- 任何包含空格的路径都必须用引号包裹
- Windows命令行使用双引号 `\"` 包裹参数
- 与原代码格式保持一致（第943-954行）

### 8.5 TeamSelectionWindow（WPF版如需批量选择）列数据精确复制

### 8.6 道具信息字符串格式

**格式定义**：
```
{道具名称}-pak:{pak文件名},mod:{bml文件名},eff:{特效代码},server:{服务器类型}
```

**示例**：
```
粉色嘻哈发型-pak:item762.pak,mod:i50124771.bml,eff:特效编号:1940,server:me
```

**各字段说明**：
- 道具名称：如"粉色嘻哈发型"
- pak文件名：如"item762.pak"（*747.pak → item747.pak）
- bml文件名：如"i50124771.bml"（从ItemCode列读取）
- 特效代码：如"特效编号:1940"或"无"
- 服务器类型："me"或"other"

**构造代码**（TeamSelectionWindow（WPF版如需批量选择）.cs）：
```csharp
string modFile = itemCode.Replace("u", "i").Replace(".png", "").Replace(".bml", "");
if (!modFile.StartsWith("i"))
{
    modFile = "i" + modFile;
}
modFile = modFile + ".bml";

string itemInfo = string.Format("{0}-pak:{1},mod:{2},eff:{3},server:{4}",
    itemName,                    // 道具名称
```
