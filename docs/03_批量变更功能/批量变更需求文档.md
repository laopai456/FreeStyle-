# 批量变更需求文档

## 一、需求概述

当前程序只支持**单对单变更**（一个道具变更前 → 一个道具变更后），用户需要支持**批量变更**功能，可以一次性完成多个道具的变更操作。

---

## 二、核心需求

### 1. 批量添加变更前道具
- **需求描述**：用户可以搜索并添加多个道具作为"变更前"的道具组
- **操作方式**：点击"添加变更前"按钮，选择道具，道具被添加到"变更前列表"中
- **界面位置**：放在"其他资源包"复选框的旁边
- **示例**：用户搜索"上衣"，添加3件上衣到变更前列表

### 2. 批量添加变更后道具
- **需求描述**：用户可以搜索并添加多个道具作为"变更后"的道具组
- **操作方式**：点击"添加变更后"按钮，选择道具，道具被添加到"变更后列表"中
- **界面位置**：放在"其他资源包"复选框的旁边
- **示例**：用户搜索"裤子"，添加3条裤子到变更后列表

### 3. 执行批量变更
- **需求描述**：将变更后的道具组，按顺序依次应用到变更前的道具组上
- **变更规则**：
  - 变更前第1个道具 → 变更后第1个道具
  - 变更前第2个道具 → 变更后第2个道具
  - 变更前第3个道具 → 变更后第3个道具
  - ...以此类推
- **示例**：
  - 变更前：[上衣A, 上衣B, 上衣C]
  - 变更后：[上衣X, 上衣Y, 上衣Z]
  - 变更结果：上衣A→上衣X, 上衣B→上衣Y, 上衣C→上衣Z

### 4. 一次性批量变更
- **需求描述**：用户一次性添加多个变更前道具和多个变更后道具，点击一次"批量变更"按钮，完成所有道具的变更
- **使用场景**：
  - 搜索添加上衣A、裤子A、鞋子A到变更前
  - 搜索添加上衣X、裤子X、鞋子X到变更后
  - 点击一次"批量变更"按钮，同时完成3个道具对的变更

### 5. 界面改造
- **原"还原全部"按钮**：改名为"批量变更"按钮
- **原"还原全部"代码**：注释掉不删除，保留代码以备后续使用
- **新增按钮**：
  - "添加变更前"按钮
  - "添加变更后"按钮
- **新增列表**：
  - "变更前列表"（显示已添加的变更前道具）
  - "变更后列表"（显示已添加的变更后道具）
- **按钮位置**：放在"其他资源包"复选框的旁边

// 变更前道具列表（存储道具信息字符串）
List<string> beforeList = new List<string>();

### 3.3 操作流程

#### 步骤1：添加变更前道具
```
用户点击"添加变更前"按钮
    ↓
弹出道具选择窗口（复用现有的表格搜索界面）
    ↓
用户选择道具，点击"添加到变更前"
    ↓
道具被添加到变更前列表
    ↓
重复以上步骤，添加多个道具
```

#### 步骤2：添加变更后道具
```
用户点击"添加变更后"按钮
    ↓
弹出道具选择窗口
    ↓
用户选择道具，点击"添加到变更后"
    ↓
道具被添加到变更后列表
    ↓
重复以上步骤，添加多个道具
```

#### 步骤3：验证
```
用户点击"批量变更"按钮
    ↓
验证条件：
1. 变更前和变更后列表都不能为空
2. 变更前和变更后的数量必须相等
    ↓
如果不满足，弹出提示对话框
```

#### 步骤4：执行批量变更
```
从变更前和变更后列表中依次取道具
    ↓
循环执行（i 从 0 到 beforeList.Count - 1）：
    1. beforeItem = beforeList[i]
    2. afterItem = afterList[i]
    3. 执行变更逻辑（复用现有的 bw_DoWorkLast 方法）
    4. 更新进度条
    ↓
所有变更完成后显示成功消息
```

#### 步骤5：清空列表（可选）
```
用户点击"清除"按钮
    ↓
清空变更前或变更后列表
```

### 3.4 交互细节

#### 添加道具时的选项
- **是否它服**：添加变更后道具时，可以选择"其他资源包"复选框
- **查看道具**：添加前可以先查看道具图标（复用现有的查看功能）

#### 列表操作
- **删除选中**：删除列表中选中的道具
- **清空列表**：清空整个列表
- **查看道具**：双击列表中的道具，可以查看图标

#### 进度显示
- **批量变更时**：显示进度条（例如："3/5" 表示已完成3个，共5个）
- **错误处理**：如果某个道具变更失败，记录错误但继续执行下一个
