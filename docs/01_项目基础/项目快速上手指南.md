# 项目快速上手指南

## 第一步：理解项目

这是一个 **FS街头篮球游戏** 的**服装搭配修改工具**，主要功能是：
- 查看游戏道具的图标
- 将一个道具的外观替换为另一个道具
- 支持本服和它服道具互换

## 第二步：快速理解代码逻辑

### 核心流程（3句话）
1. **查看道具** → 复制pak文件 → 解包 → 显示图标
2. **设置变更前后** → 记录道具信息 → 准备替换
3. **执行变更** → 复制模型文件 → 重新打包

### 代码文件优先级
1. **先看** `完整操作逻辑梳理.md` - 了解代码层面的操作流程
2. **再看** `核心代码文件索引.md` - 快速定位关键方法
3. **最后** `调试监听器说明.md` - 了解如何使用调试日志

## 第三步：运行程序

### 编译
在 VS Code 中按 `Ctrl+Shift+P` → 选择 `.NET: Build`

### 准备环境
```
FS服装搭配专家v5.3.6\bin\Debug\
├── FS服装搭配专家v5.3.6.exe  # 编译生成的EXE
├── pack\
│   └── resources.exe             # 必须复制此文件
└── logs\                        # 调试日志会生成在这里
```

### 运行
直接运行 `bin\Debug\FS服装搭配专家v5.3.6.exe`

## 第四步：使用调试日志

程序运行后，所有操作都会记录到 `logs\operation_debug.log`

查看日志了解：
- 用户点击了什么
- 程序执行了哪些步骤
- 哪里出错了

## 第五步：修改代码

### 添加新功能
1. 参考 `完整操作逻辑梳理.md` 了解现有逻辑
2. 参考 `核心代码文件索引.md` 找到相关方法
3. 修改代码，添加调试日志
4. 编译测试
5. 查看日志确认逻辑正确

### 常见修改场景
- **修改文件路径**：搜索 `Environment.CurrentDirectory`，改为 `Application.StartupPath`
- **添加日志**：调用 `Elena.Debugger.OperationDebugger.LogXXX()`
- **修改对话框**：搜索 `MessageBox.Show()`

## 第六步：遇到问题

1. 查看调试日志
2. 查阅 `常见问题FAQ.md`
3. 查阅 `完整操作逻辑梳理.md` 理解逻辑
