### 实测修复的问题

| 问题 | 根因 | 修复 |
|---|---|---|
| 共享内存找不到 | DLL 用 `Global\` 前缀创建文件映射，目标进程（非管理员）无 `SeCreateGlobalPrivilege` | 改为 session-local 名 `HwBpVEHSharedMem` |
| Python 读共享内存拒绝访问 | ctypes 未设置 `restype = c_void_p`，`MapViewOfFile` 返回的 64 位指针被截断 | 显式设 `restype` |
| Python 读共享内存拒绝访问 | Python struct 天然对齐 (128 bytes) vs C `pack(1)` (108 bytes)，请求的 mapping 大小超过实际 | 加 `_pack_ = 1` |
| IOCTL 失败错误 1 | Python 客户端 IOCTL 编码用了 Method=3，驱动用 Method=0 | 修正为 `(0x22 << 16) \| (func << 2)` |

## 卡住 🚧

驱动服务能启动（`sc query` → RUNNING），但设备节点 `\\.\HwBpDriver` 打不开（错误 2 → 文件不存在）。

```
sc start HwBpDriver   → STATE: 4 RUNNING
CreateFile(\\.\HwBpDriver) → 错误 2 (ERROR_FILE_NOT_FOUND)
```

原因未锁定，两种可能：

**A. 驱动签名问题** — 驱动是测试签名证书，需要系统开启测试签名模式
```
bcdedit /set testsigning on
```
重启后生效。

**B. 驱动崩溃后内核残留** — 之前 IOCTL 错误请求打崩驱动后，`sc delete` + `sc create` 重建也没清干净，需要重启清除内核残留。

最有把握的排除方法：**重启电脑**，然后 `sc start HwBpDriver`。
