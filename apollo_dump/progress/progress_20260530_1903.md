# §107 练习场崩溃 — 静态/动态Actor骨骼不兼容

> ⏳ **已归档** — 信息已提取到 01_知识库 / 02_试验记录 / 03_常量地址表


> 2026-05-30 §106方案渲染成功，但进入练习场时崩溃

## 关键结论

**渲染成功，但崩溃**: sprintf hook + BML方案在第一场成功渲染紫色超赛发型。进入练习场（第二次加载）时硬崩溃。

**根因**: 美丽梦想(50125461)是DStaticActor，紫色超赛(50125711)需要DDynamicActor的物理骨骼。第一场勉强渲染，练习场动画系统访问不存在的物理骨骼 → 崩溃。

## 试验记录

### §107.1 BML路径修正后首次成功
- **结果**: ✅ 渲染成功（第一场）
- **关键修正**: BML放到 `Resource\item768\` 而非 `Resource\item\`（CreateFileA hook确认）
- **BML内容**: 从item768.pak提取的完整XML(1573字节)有效

### §107.2 练习场崩溃
- **结果**: ❌ 硬崩溃
- **现象**: 
  - 第二次sprintf替换(#2)触发
  - AcquireSMD加载 `res768\i50125711_FN.smd`(269824字节)
  - 继续加载其他角色（#3, #4替换）
  - ReadFile到#164时崩溃
- **根因**: DStaticActor不支持DDynamicActor的物理骨骼动画

## 已验证的完整调用链

```
sprintf("i%d.xml", 50125461) → 替换为50125711
  → [ebp-0xD8]也改
  → LoadItemFile(50125711, "i50125711.xml")
  → Group="item768" (binary实际行为)
  → SeekFile: Resource\item768\i50125711.bml ← 松散文件命中
  → LoadBinaryXML: XOR 0xFF解码 → XML
  → 解析出SMD路径 (res768\i50125711_*.smd)
  → AcquireSMD → AcquirePack("res768") → 加载res768.pak
  → 紫色超赛发型渲染成功! (第一场)
  → 练习场 → 崩溃 (骨骼不兼容)
```
