# §105 文件打开函数诊断 — Group机制揭示

> ⏳ **已归档** — 信息已提取到 01_知识库 / 02_试验记录 / 03_常量地址表


> 2026-05-30~31 sprintf+ebp-D8补丁后仍光头，追查BML查找失败根因

## 关键结论

**根因确认**: sprintf替换+ebp-D8修复后，LoadItemFile收到正确的iItemCode=50125711，但内部文件打开函数(DFileGPack::Open)查找BML失败。原因是游戏只搜索`item.pak`和`item_ext.pak`，而i50125711.bml在`item768.pak`中。

**修复路径**: 将XOR编码的BML放到`Resource\item\i50125711.bml`，SeekFile先查松散文件即可命中。

## 试验记录

### §105.1 sprintf+[ebp-0xD8]双重替换验证
- **方法**: hook_sprintf_replace.py v2，同时改sprintf参数和[ebp-0xD8]
- **结果**: ✅ 替换成功
  - sprintf输出: i50125711.xml ✓
  - [ebp-0xD8]: 50125461→50125711 write_ok=True ✓
  - LoadItemFile收到: iItemCode=50125711 ✓
- **问题**: 角色仍光头，AcquireSMD未加载50125711的SMD

### §105.2 0x1ACDF33 hook (函数中间hook — 崩溃)
- **方法**: hook_diag_fileopen.py，hook 0x1ACE312(函数中间地址)和0x1ACDF33
- **结果**: ❌ 崩溃
  - Frida Interceptor不支持中间地址hook
  - 参数全是垃圾值(ic=2243999247)
  - 正常LoadItemFile全部中断，只有1次垃圾调用
- **根因**: 0x1ACE312是LoadItemFile内部的`movzx eax,al`指令，不是函数入口

### §105.3 0x1ACDF33 hook (只入口hook — 仍崩)
- **方法**: hook_diag_fileopen_v2.py，只hook函数入口0x1ACDF33
- **结果**: ❌ 仍崩
  - FileOpen #1参数仍是垃圾(ic=3921720847)
  - 代码dump显示Frida已patch前5字节(e9=JMP)
  - return address计算异常(负值)
- **结论**: 0x1ACDF33不可hook，Frida trampoline对此函数不兼容

### §105.4 只读Group诊断 (安全hook)
- **方法**: hook_diag_group_read.py，只hook LoadItemFile入口(安全)，读ecx->0x7E4/0x7E8
- **结果**: ✅ 成功获取Group信息
  - `this->0x7E4 = 0x320 (800)` — flag==0时的Group
  - `this->0x7E8 = 0x321 (801)` — flag!=0时的Group
  - **所有item的Group完全相同**，包括替换后的50125711
  - Group=800是**角色编号**(c800)，不是item的PAK编号

### §105.5 源码确认BML查找机制
- **方法**: 读取源码 DFileGPack.cpp + Character.cpp
- **结果**: ✅ 关键发现
  - LoadItemFile构造: `SFullName("item", "i50125711.bml")` — Group="item"硬编码
  - SeekFile搜索顺序: `Resource\item\i50125711.bml`(松散) → `item.pak` → `item_ext.pak`
  - **item768.pak不在搜索范围内**
  - AcquirePack("item")只加载item.pak

## 关键发现

1. **Group=0x320(800)是角色组号，不是item PAK号**。所有item共享同一个Group，由角色c800决定
2. **BML查找路径固定**: Group永远是"item"(字符串)，不是整数800
3. **0x1ACDF33(DFileGPack::Open)不可用Frida hook** — 会崩溃，无论入口hook还是中间hook
4. **BML在item768.pak但游戏只查item.pak** → 找不到 → 光头
5. **修复方案**: 松散文件 `Resource\item\i50125711.bml`，SeekFile优先查松散文件
6. **BML需XOR编码**: LoadBinaryXML总是做XOR 0xFF解码，松散文件必须是编码版

| LoadItemFile | 0x1ACE1C0 | 加载item BML，thiscall(ecx=this, arg1=iItemCode) |

### DFileGPack::Open 查找逻辑 (源码)
```cpp
// 1. 松散文件
path = working_path + "Resource\\" + Group + "\\" + Name;
if (file_exists(path)) return true;

// 2. 主PAK
AcquirePack(Group);  // 加载 Group + ".pak"
if (binary_search_in_pak(Name)) return true;

// 3. 扩展PAK
AcquirePack(Group + "_ext");  // 加载 Group + "_ext.pak"
if (binary_search_in_pak(Name)) return true;

return false;
```

### LoadItemFile BML构造 (源码)
```cpp
SString sFilename = DCutPath(DCutFileExt(szFilename)); // "i50125711"
SFullName nameItem("item", sFilename + ".bml");         // Group="item"
DFileGPack packItem;
packItem.Open(nameItem);  // → SeekFile 查找
```
