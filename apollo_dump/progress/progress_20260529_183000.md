### hook_acquire_caller.py — 调用者 (0x1EEBA30) 分析
ecx(this)对象 SString 布局:
```
+0x00: 序号 (0,1,2...12)
+0x04: 相关指针
+0x08: 0x00
+0x0C: 栈/堆指针
+0x10: 字符串数据指针 ← SMD文件路径
+0x14: 字符串长度
+0x18: 容量(=长度)
+0x1C: flags (0或1)
```

## §210: 替换实验 — 三轮迭代

### v1: hook_acquire_replace.py — 崩溃
Bug: ecx+0x10是指针, 脚本直接往ecx+0x10写数据 → 覆盖了指针 → 对象损坏 → 崩溃。

### v2: hook_acquire_replace2.py — 误匹配
修复: 跟随指针 ptr(ecx+0x10.readU32()) → 写入堆字符串。
新Bug1: `res767`匹配太宽, 把其他物品的pak路径也改了(i50519561)。
新Bug2: 很多调用的指针在0x8xxxxxxx范围, 被`>= 0x80000000`过滤。

### v3: hook_acquire_replace3.py — 替换成功但游戏崩溃
修复1: 精确匹配 — 只在包含ItemCode 50125461时才替换。
修复2: LARGEADDRESSAWARE — FreeStyle堆地址可到0x8xxxxxxx, 去掉上界检查。
结果:
```
#10: "res767\i50125461_FN.smd" → "res768\i50125711_FN.smd" (2处替换)
```
SSKF加载正常(36个文件), 但进入房间后游戏崩溃。
**崩溃发生在mesh加载之后, 推测是渲染/骨骼不兼容。**

## §211: 关键发现

### FreeStyle是LARGEADDRESSAWARE
堆分配地址可达0x8xxxxxxx, 指针检查不能以0x80000000为上界。
v2的日志中#3-#12全是0x8xxxxxxx指针, 全被过滤, 导致目标文件没出现。

### 替换链路已打通
```
调用者入口 0x1EEBA30
  → ecx+0x10 读指针 → 堆字符串 "res767\i50125461_FN.smd"
  → 原地替换字节 → "res768\i50125711_FN.smd"
  → AcquireSMD 0x01EEC130 执行
  → ReadFile 读到SSKF数据 (36个文件正常)
  → 后续处理崩溃 (骨骼/动画不兼容)
```

### 数据兼容性问题
- 美丽梦想发型(50125461): 静态发型, 无物理骨骼
- 紫色超赛发型(50125711): 动态发型, 有物理模拟
- 直接替换SMD文件名可行, 但mesh数据中的骨骼引用导致崩溃
