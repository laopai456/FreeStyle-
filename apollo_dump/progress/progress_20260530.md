## §303: 突破性方案 — sprintf ItemCode替换

### 原理

源码 Character.cpp:
```cpp
// SetCharacterFeature 内部循环:
sprintf(szItem, "customize\item\i%d.xml", iItem);  // ← hook这里
LoadItemFile(iItem, szItem);                         // szItem被处理成BML路径

// LoadItemFile 内部:
SString sFilename = szFilename;
sFilename = DCutPath(sFilename);    // "i50125711.xml"
sFilename = DCutFileExt(sFilename); // "i50125711"
SFullName nameItem("item", sFilename + ".bml"); // "item\i50125711.bml"
// → 从PAK加载50125711的完整BML → 解析出50125711的mesh路径
```

### 替换策略

**Hook MSVCR100 sprintf**, 当:
- 格式串 = `"customize\item\i%d.xml"`
- args[2] (ItemCode) = 50125461

替换 args[2] 为 50125711。

### 效果链

```
sprintf输出: "customize\item\i50125711.xml"
  → DCutPath: "i50125711.xml"
    → DCutFileExt: "i50125711"
      → SFullName("item", "i50125711.bml")
        → PAK加载50125711的BML (含完整mesh路径)
          → AcquireSMD加载50125711的SMD (骨骼+物理)
            → 角色显示紫色超赛发型!
```

### 关键优势

1. **骨骼兼容** — 50125711的BML+mesh+骨骼是完整配套的
2. **服务器无感** — `pItem->iItemCode`仍为50125461 (LoadItemFile的第一个参数)
3. **单点拦截** — 只hook sprintf一个函数
4. **可扩展** — 可以替换任何物品

### 待验证

- 50125461是静态发型(无自己SMD), 50125711是动态(9个SMD)
- BML格式不同(.bm vs .bml), LoadItemFile是否能正确解析
- 需要确认res768.pak中有50125711的BML条目
