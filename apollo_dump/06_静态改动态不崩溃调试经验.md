## 二、关键发现

### 2.1 资源完整性是前提

**问题**：游戏的 `res768.pak` 里 50125711 的资源不完整，只有 `i50125711_fn.smd`。

**解决**：从源文件目录 `C:\Users\w\Desktop\res768_pak` 部署完整资源到 `Resource\res768\`。

**教训**：
> 替换前必须确认目标发型的资源是否完整存在于游戏 PAK 中。如果 PAK 里资源不完整，需要从外部补齐。

---

### 2.2 部署目录必须正确

**错误**：`deploy_clean_bml.py` 最初部署到 `Resource\item\`

**正确**：部署到 `Resource\item768\`

**原因**：DFileGPack 查找链是：
```
group=item768 → Resource\item768\{name}.bml → item768.pak\{name}
```

**教训**：
> 部署目录必须和目标 ItemCode 的 group 一致。group 从 ItemCode 推断：`item{编号前三位}`。

---

### 2.3 Apollo 干扰需要排除

**问题**：`Process.setExceptionHandler` 会捕获 Apollo 抛出的 `INT3` 异常，干扰阶段机。

**解决**：设置 `ENABLE_EXCEPTION_HANDLER = False`，禁用异常处理器。

**教训**：
> 调试阶段先禁用异常处理器，排除 Apollo 干扰，确认业务逻辑正常后再开启。

---

### 2.4 `[ebp-0xD8]` 同步修改是必要的

**问题**：只替换 `sprintf` 的 ItemCode 参数，`LoadItemFile` 仍拿到原始值。

**解决**：`PATCH_EBP_D8 = True`，同步修改调用栈上的 ItemCode。

**教训**：
> `sprintf` 替换只是第一步，后续函数可能从别的地方读参数。需要追踪完整调用链。

---

### 2.5 CreateFileA/W 监听对 PAK 内资源无效

**问题**：BML 被 CreateFile 看到，但 SMD/PNG 看不到。

**原因**：SMD/PNG 在 PAK 内部，游戏直接从 PAK 读取，不走 CreateFile。

**教训**：
> PAK 内资源的读取不经过 CreateFile，需要 hook 游戏内部的 PAK 解析函数或 AcquireSMD 调用点。

---

## 三、当前遗留问题

### 3.1 练习场恢复默认发型

**现象**：
- 房间：目标发型正确显示
- 练习场：恢复默认发型
- 退出练习场：又变回目标发型

**原因**：练习场阶段走了不同的加载逻辑，sprintf 替换没有覆盖到。

**下一步**：需要 hook 练习场阶段的角色加载函数。

---

## 四、调试流程总结

### 4.1 资源层验证

1. 确认目标 ItemCode 的资源是否完整
2. 确认部署目录是否正确（`Resource\item{group}\` 和 `Resource\res{group}\`）
3. 用 `verify_bml.py` 验证 BML 引用的资源路径

### 4.2 Hook 层验证

1. 确认 `sprintf` 替换是否触发
2. 确认 `CreateFileA/W` 是否看到 BML 文件
3. 确认游戏是否读取到正确的 BML

### 4.3 运行层验证

1. 禁用异常处理器，排除 Apollo 干扰
2. 观察各阶段（房间/练习场/大厅）的发型状态
3. 确认是否崩溃

## 五、关键代码片段

### 5.1 sprintf hook + ebp-0xD8 同步修改

```python
# Python 配置
SRC_IC = 50125461
DST_IC = 50125711
PATCH_EBP_D8 = True
```

```javascript
// JS hook
Interceptor.attach(sprintfAddr, {
    onEnter: function(args) {
        var fmt = readAscii(args[1], 80);
        if (fmt.indexOf('customize\\item\\i%d.xml') >= 0) {
            var itemCode = args[2].toInt32();
            if (itemCode === SRC_IC) {
                args[2] = ptr(DST_IC);
                if (PATCH_EBP_D8) {
                    this.context.ebp.sub(0xD8).writeU32(DST_IC);
                }
            }
        }
    }
});
```
